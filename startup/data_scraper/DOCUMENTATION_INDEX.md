## 📱 Documentation Format Guide

### Structure of MODULE_GUIDE.md

For each module, you'll find:

1. **Purpose** - What it does
2. **Usage** - How to run it
3. **Input File Requirements** - Required CSV/Excel columns
4. **Command-Line Arguments** - All flags and options
5. **Output Format** - What you get back
6. **Examples** - Real command examples
7. **Performance Tips** - Speed and optimization
8. **Troubleshooting** - Common issues

### Example Entry:
```markdown
### Module Name

**File**: `src/module.py`

#### Purpose
[What it does]

#### Usage
[How to run it]

#### Input Requirements
[CSV columns needed]

#### Arguments
[All --flags]

#### Examples
[Real commands]
```
